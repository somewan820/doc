# 商品价值设计规约

## 一、目标

本规约用于指导 AI 或策划设计新的商品数据，确保商品的属性、价值分、稀有度和价格之间保持一致。

商品不按法宝、丹药等类别区分价值倍率。只要最终属性表现相同，商品价值和价格就应相同。

## 二、商品字段

每个商品只保留以下字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `itemId` | string | 是 | 商品唯一 ID，建议使用语义化前缀 |
| `displayName` | string | 是 | 玩家可见名称 |
| `Unlocked By Default` | boolean | 是 | 是否默认解锁，默认填 `false` |
| `value` | number | 是 | 综合价值分，由三种属性计算得出 |
| `attack` | int | 是 | 攻击属性，不能为负数 |
| `defense` | int | 是 | 防御属性，不能为负数 |
| `movementSpeed` | int | 是 | 移动速度属性，不能为负数 |
| `rarity` | enum/string | 是 | 商品稀有度 |
| `description` | string | 是 | 商品描述，说明外观、来历、效果或氛围 |
| `price` | int | 是 | 商品价格，由 `value` 和 `rarity` 计算得出 |

示例：

```json
{
  "itemId": "fabao_qingfengjian",
  "displayName": "青锋剑",
  "Unlocked By Default": false,
  "value": 3,
  "attack": 3,
  "defense": 0,
  "movementSpeed": 0,
  "rarity": "Common",
  "description": "最普通的制式飞剑，铁匠铺量产之物。剑身泛着淡淡的青光，虽不锋利，但对付野兽绰绰有余。落仙城南坊的散修几乎人手一柄。",
  "price": 12
}
```

## 三、稀有度

`rarity` 只能填写以下五档：

| 稀有度 | 定位 | 建议 `value` 区间 |
| --- | --- | --- |
| `Common` | 入门、廉价、凡俗或低阶修士常见物 | `1` 到 `5` |
| `Fine` | 有明确灵性或战斗用途，适合筑基前后使用 | `6` 到 `18` |
| `Superior` | 高阶精品，适合金丹层级或重要剧情奖励 | `25` 到 `60` |
| `Epic` | 镇店、传承、稀世宝物，适合元婴层级 | `100` 到 `190` |
| `Immortal` | 仙品、上古神物、版本顶级追求 | `220` 以上 |

现有商品迁移关系：

| 原稀有度 | 新稀有度 |
| --- | --- |
| `Common` | `Common` |
| `Uncommon` | `Fine` |
| `Rare` | `Superior` |
| `Legendary` | `Epic` |

`Immortal` 是最高预留档，不应随意使用。只有当商品明显强于 `Epic`，并且在剧情或玩法中承担顶级奖励定位时，才使用 `Immortal`。

## 四、价值计算

`value` 由三种属性计算得出：

```text
valueRaw = attack + defense + movementSpeed * 0.8
value = round(valueRaw, 1)
```

规则说明：

- `attack` 和 `defense` 每 1 点计为 1 点价值。
- `movementSpeed` 每 1 点计为 0.8 点价值。
- `value` 保留 1 位小数。
- 如果 `value` 正好是整数，可以直接写整数，例如 `3`，不必写成 `3.0`。
- 三种属性必须是非负整数。

## 五、价格计算

`price` 由 `value` 和 `rarity` 计算：

```text
price = round(value * rarityPriceFactor)
```

各稀有度价格系数：

| 稀有度 | 价格公式 |
| --- | --- |
| `Common` | `price = round(value * 4)` |
| `Fine` | `price = round(value * 6.5)` |
| `Superior` | `price = round(value * 10.5)` |
| `Epic` | `price = round(value * 16)` |
| `Immortal` | `price = round(value * 24)` |

设计原则：

- 同一稀有度内，`value` 越高，`price` 必须越高。
- 稀有度越高，单位 `value` 的价格越高。
- 不因商品类型改变价格，同属性、同稀有度的商品价格应一致。
- 高速度商品可以更贵，但速度已经通过 `0.8` 权重折算，不要额外加价。

## 六、AI 设计新商品步骤

设计新商品时按以下顺序执行：

1. 先确定商品定位和 `rarity`。
2. 按稀有度选择合理的目标 `value` 区间。
3. 根据商品设定分配 `attack`、`defense`、`movementSpeed`。
4. 用公式计算 `value`。
5. 用对应稀有度价格系数计算 `price`。
6. 检查同档商品，避免低 `value` 商品比高 `value` 商品更贵。
7. 编写能解释属性倾向的 `description`。

## 八、常见错误

错误示例：

```json
{
  "itemId": "bad_item",
  "displayName": "错误商品",
  "Unlocked By Default": false,
  "value": 50,
  "attack": 1,
  "defense": 1,
  "movementSpeed": 1,
  "rarity": "Epic",
  "description": "属性与价值不匹配。",
  "price": 300
}
```

问题：

- 属性实际价值为 `1 + 1 + 1 * 0.8 = 2.8`，不能写成 `50`。
- `Epic` 的价格应为 `round(value * 16)`。
- 价格不能脱离公式手动填写。

正确写法：

```json
{
  "itemId": "fixed_item",
  "displayName": "修正商品",
  "Unlocked By Default": false,
  "value": 2.8,
  "attack": 1,
  "defense": 1,
  "movementSpeed": 1,
  "rarity": "Common",
  "description": "属性很低，只适合作为入门商品。",
  "price": 11
}
```
