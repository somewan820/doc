# 整体剧情流程图（经营叙事版）

## 一、整体结构

```mermaid
flowchart TD
    A[序章开场<br/>南坊开门 店铺重新营业] --> B[玩家进入日常经营循环]
    B --> C[阶段性来客进入店铺]
    B --> D[核心NPC 陆沉第一次进店]

    C --> C1[单次来店]
    C1 --> C2[系统给出本轮提示词]
    C2 --> C3[大模型生成<br/>NPC开场话 + 玩家三个回复 + NPC延展话]
    C3 --> C4[完成本轮闲聊与购物]
    C4 --> C5[该NPC离店]
    C5 --> B

    D --> D1[Round 1<br/>第一次进店]
    D1 --> D2[想买便宜耐用的护体法宝]
    D2 --> D3[玩家与他第一次闲聊]
    D3 --> D4[留下第一印象<br/>沉默 克制 临时停靠]
    D4 --> E1

    E1[Round 3<br/>第二次来店] --> E2[试着出城又折返]
    E2 --> E3[买回气丹和轻身符]
    E3 --> E4[玩家察觉 他没走成]
    E4 --> F1

    F1[Round 6<br/>第三次来店] --> F2[像熟客一样进门看货]
    F2 --> F3[补护体丹 看旧法宝]
    F3 --> F4[关系变熟 但仍未留下]
    F4 --> G1

    G1[Round 10<br/>第四次来店] --> G2[清晨进店 真正准备离城]
    G2 --> G3[补最后一程的路上用品]
    G3 --> G4[弱主线收束<br/>终于继续上路]
    G4 --> H[回到经营主循环]

    H --> I[更多阶段性来客持续进入]
    I --> J[玩家通过经营与闲聊见证众生]
```

## 二、单轮玩法循环

```mermaid
flowchart LR
    A[某NPC来店] --> B[事件系统给出当前事件text]
    B --> C[从text中取出<br/>上一轮提示词 + 本轮提示词 + 回复方向 + 店铺上下文]
    C --> D[大模型生成对话]
    D --> E[NPC开场]
    E --> F[玩家三个回复]
    F --> G[NPC继续讲述]
    G --> H[完成一轮购物/闲聊]
    H --> I[事件系统写入下一事件ID]
    I --> J[等待下次来店]
```

## 四、阅读版简图

```text
序章
  -> 南坊开门
  -> 玩家接手日常经营
  -> 第一个重要来客出现

日常经营主循环
  -> 来客进店
  -> 事件系统给出本轮提示词
  -> 大模型生成闲聊
  -> 玩家回复
  -> NPC继续讲述
  -> 完成购买/离店
  -> 写入下一轮事件

核心NPC（陆沉示例）
  -> 第一次来店：临时停靠，只买够继续走下去的东西
  -> 第二次来店：试着离城又折返
  -> 第三次来店：像熟客一样回来看货
  -> 第四次来店：真正准备上路
  -> 收束：他离开，但店铺留在他的路上

阶段性NPC
  -> 不承担强主线
  -> 负责南坊烟火气和世界横截面
  -> 每个人只提供一小段人生切面
```

## 五、这一版剧情的核心感受

不是：

- 玩家破解大案
- 玩家打出最优分支
- 所有NPC互相咬合推进世界结局

而是：

- 玩家守着店
- 看一些人带着各自的问题进门
- 通过一次次来店，看出他们慢慢变了
- 最后意识到，自己见证了一段路
